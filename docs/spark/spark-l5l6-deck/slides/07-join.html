Join 操作

Four Types + Broadcast
两张表按共同 key 拼接 —— Join 也会触发 Shuffle，除非广播小表。
01 / INNER
inner
只保留两边都有的
02 / LEFT
left
保留左表所有行，右表缺则 null
03 / RIGHT
right
保留右表所有行
04 / FULL
full outer
两边都保留
broadcast_join.py
from pyspark.sql.functions import broadcast # orders（100亿行）JOIN users（5000行）—— 小表直接广播 fast = orders.join(broadcast(users), on="user_id", how="left") # 自动广播阈值默认 10MB spark.conf.set("spark.sql.autoBroadcastJoinThreshold", "50m")
BROADCAST 小表 broadcast 到每个节点 → 大表零 Shuffle，性能提升巨大。
